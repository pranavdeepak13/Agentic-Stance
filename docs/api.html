API Reference

Public functions, classes, and dataclasses for all simulation modules. Internal helpers prefixed with _ are omitted.

config.py

src/config.py
All data types, SimulationConfig, config loading, and stance parsing. Every other module imports its types from here. This is the single source of truth for enums, exceptions, and constants.

Functions

LikertStance.from_score(score: int) → LikertStance
Reverse lookup — returns the LikertStance member whose .score equals score. Used by the annotator after clamping to convert a clamped integer back to a stance member.
RaisesValueError if score is not one of -2, -1, 0, 1, 2.
load_config() → SimulationConfig
Reads all env vars, validates all fields, smoke-tests every prompt template. Fails immediately with a full list of errors — never silently mid-run. Calls _check_ollama() or _check_vllm_config() depending on provider.
RaisesValueError on any config error. LLMError if Ollama is unreachable or model is missing.
parse_likert(text: str) → LikertStance
Parses a Likert label from LLM output. Strategy: (1) exact case-insensitive match on the full stripped response; (2) longest-label substring match so "Strongly Against" is matched before "Against" when both appear in the text.
RaisesAnnotationError if no label is found anywhere in the text.

src/personas.py
Loads agent personas from topic modules. The topic file is the only place persona data lives — this module is a loader, not a store.

llm.py

src/llm.py
Unified async LLM access with concurrency-bounded batch dispatch. Supports ollama, anthropic, openai, groq, vllm. All providers implement the same interface through _bounded_gather().
dataclassLLMRequest (frozen)
FieldType
systemstr
userstr
asyncllm_call(system: str, user: str, config: SimulationConfig) → str
Convenience wrapper — calls llm_call_many() with a single request. Use for one-off calls in exchange.py.
RaisesLLMError on provider failure.
asyncllm_call_many(requests: list[LLMRequest], config: SimulationConfig) → list[str]
Dispatches all requests concurrently, bounded by config.llm_max_concurrency. Results are aligned with the input list order. For vLLM, the server batches concurrent requests internally — the application-level contract does not change.
RaisesLLMError on provider failure. Individual request failures propagate through the gather.

asyncrun_exchange(agent_a, agent_b, n_turns, current_stances, memory, config, sim_clock, day=0) → list[ExchangeTurn]
Runs a multi-turn dialogue between agent_a and agent_b. Agents alternate as speaker/listener. After each turn the listener absorbs the utterance via memory.absorb(). Returns the full transcript. The day parameter is passed as round=day to memory.absorb().
Returnslist[ExchangeTurn] — one entry per turn.

Runs multiple exchange jobs concurrently via asyncio.gather. Jobs must be independent — no shared mutable stance state, no overlapping participants. Used by the parallel vLLM scheduler path in simulation.py.
ReturnsList of exchange logs aligned with the input jobs list.

annotator.py

src/annotator.py
Stance annotation by a third-party LLM judge. The annotator reads an agent's utterances from a completed exchange and returns a LikertStance. It is entirely separate from the agent LLMs — it sees the exchange as a reader, not a participant.
The annotator clamps its output to ±1 from the agent's previous stance. This is applied in both the batch path and the retry path. MAX_RETRIES = 3.
asyncannotate_many(items: list[tuple[AgentPersona, str, LikertStance]], config) → list[LikertStance]
Batch annotation. Each item is (agent, utterances_text, previous_stance) where utterances_text is the pre-filtered string of only that agent's utterances. Fires all LLM requests in a single llm_call_many() call, then clamps each result to ±1 from previous_stance. Falls back to _retry_annotation() on AnnotationError.
ReturnsList of LikertStance aligned with the input list.
asyncannotate(agent, exchange_log, previous_stance, config) → LikertStance
Single-agent wrapper around annotate_many(). Filters utterances from exchange_log to only the target agent's turns, then calls the batch function. Returns previous_stance if the agent did not speak (should not happen in normal operation).

memory.py

src/memory.py
AgentMemory interface and two concrete implementations. exchange.py calls absorb() and get_context() without knowing which implementation is active — the memory condition lives only in simulation.py.
ABCAgentMemory
Abstract base class. Defines the two methods every memory implementation must provide.
MethodSignature
absorbasync (agent_id, text, author, round=0) → None
get_context(agent_id, topic) → str
classNullMemory(AgentMemory)
No-op implementation for the no_kg condition. Both absorb() and get_context() are no-ops that return None and "" respectively. Zero conditionals in exchange.py — the Null Object Pattern means the loop never checks the condition.
classGhostMemory(AgentMemory)
KG-backed implementation. Wraps a ghost_kg.AgentManager. The dimensions list controls which KG dimensions are active — one class serves all three KG conditions.
GhostMemory(manager, config, dimensions=["general"])         # general_only
GhostMemory(manager, config, dimensions=["tom"])             # tom_only
GhostMemory(manager, config, dimensions=["general","tom","beliefs"])  # full_kg

absorb() calls extract_triplets_batch() for each dimension, then writes triplets via manager.absorb_content(). Failures are logged as warnings — a KG write failure never crashes the simulation.

get_context() calls manager.get_context() which returns FSRS-ranked triplets. Returns empty string on failure.

src/triplets.py
Extract knowledge graph triplets from utterance text using an LLM. Returns empty list on any parsing failure — the simulation never crashes because triplet extraction failed.

FieldTypeNotes

predicatestr

roundintDay number. Default 0. Used for recency filtering on retrieval.
asyncextract_triplets(text, dimension, config) → list[Triplet]
Single-request convenience wrapper. Delegates to extract_triplets_batch(). Returns empty list on failure.
asyncextract_triplets_batch(requests: list[tuple[str, str]], config, round=0) → list[list[Triplet]]
Extracts triplets for many (text, dimension) pairs in one batch LLM call. Results are aligned with the input list. Individual failures degrade to empty lists — the batch itself does not raise. Returnslist[list[Triplet]] aligned with input requests.

tracker.py

src/tracker.py
Append-only logging of simulation results. Two output files: results.csv (stance trajectory, one row per exchange) and exchanges.jsonl (full conversation transcripts). Both are opened and closed atomically on every write.
classSimulationTracker

Constructed once in simulation.py with config and a run_id timestamp string.

SimulationTracker.log(iteration, agent_a, agent_b, stance_a_before, stance_a_after, stance_b_before, stance_b_after, exchange_log, sim_clock)
Appends one row to results.csv and one JSON object to exchanges.jsonl. Both writes are atomic — a crash between them leaves at most one extra JSONL record, which the resume logic handles by rebuilding stances from CSV only.

agent_b_stance_before_label, agent_b_stance_before_score,
agent_b_stance_after_label,  agent_b_stance_after_score,
n_turns, sim_clock

checkpoint.py

src/checkpoint.py
Crash recovery for the simulation loop. Contract: backup_db(N-1) before starting day N, write_checkpoint(N) only after tracker.log() succeeds. This guarantees SQLite state and CSV are always consistent.
write_checkpoint(iteration: int, config) → None
Writes checkpoint.json with last_completed_iteration, memory_condition, random_seed, topic. Called once per day, after all exchanges in that day complete.
load_checkpoint(config) → int | None
Returns last_completed_iteration from checkpoint.json, or None for a fresh run.
backup_db(iteration: int, config) → None
Copies simulation.db to simulation_iter_{iteration}.db.bak. Skips silently if no DB exists yet (first run, no KG condition).
restore_db_backup(last_completed: int, config) → None
Restores the SQLite DB to the state at end of last_completed. If no backup exists but a DB does, removes the DB (CSV is the source of truth for resume — the KG restarts from scratch for the resumed portion).
rebuild_stances(personas, csv_path) → dict[str, LikertStance]
Reads the full CSV and replays stance updates in order. Returns the final stance dict reflecting the last annotated stance for every agent. Called on resume to restore in-memory state from the CSV log.
Returnsdict[agent_id, LikertStance]. Falls back to initial stances if CSV does not exist.

simulation.py

src/simulation.py
Top-level orchestrator. The only module that knows which memory condition is active. Owns the current_stances dict and the main loop. All other modules are wired together here via dependency injection.
asyncrun_simulation(config: SimulationConfig) → None
Entry point. Loads personas, resumes or starts fresh, builds memory, runs the day-based loop (sequential) or parallel scheduler (when parallel_exchange_jobs > 1). The sequential path is where Phase 1 changes live — the parallel scheduler is Rossetti's vLLM addition and is left separate.
Sequential vs parallel path: When PARALLEL_EXCHANGE_JOBS == 1, the day-based loop runs. When > 1, _run_parallel_scheduler() takes over — it uses ExchangeJob batches and run_exchange_jobs() to run disjoint pairs concurrently within each batch.
