GhostKG Opinion Dynamics

Extending LLM opinion dynamics with structured decaying KG memory to measure sycophancy reduction

Collaboration: Pranav Deepak & Dr. Giulio Rossetti (CNR Italy / KDD Lab Pisa)  ·  Extends: Cau et al. 2025, EPJ Data Science (LODAS framework)

Overview

LLM agents in opinion dynamics simulations are sycophantic — they shift their stated opinion toward whoever they just spoke with, regardless of prior beliefs. This project tests whether structured knowledge graph memory (GhostKG + FSRS spaced repetition) changes that behavior.

The core claim: "Consistency-awareness mechanisms via KGs reduce intrinsic agreeableness and introduce more realistic friction into individual agent dynamics."

The system runs four ablation conditions across the same agents and topic. No-memory baseline (LODAS replication) vs. three KG configurations that give agents progressively more structured memory of past exchanges.

This is not a replication of the EPJ paper — model, topic, and population differ. Results compare conditions against the project's own no_kg baseline, not EPJ figures.

GhostKG & FSRS

GhostMemory wraps a ghost_kg.AgentManager. Each agent has its own knowledge graph keyed by agent_id. When a listener receives an utterance, absorb() extracts triplets and writes them via AgentManager.absorb_content().

The FSRS (Free Spaced Repetition Scheduler) algorithm scores each triplet's retrievability based on elapsed time since last review. The clock advances CLOCK_ADVANCE_HOURS per simulated day (default 24h). At 50 iterations that is roughly 7 weeks of simulated time — enough for early-day memories to decay substantially.

Simulation Mechanics

Day-based loop

One iteration equals one day. Every agent initiates exactly one exchange per day. Order is randomized each day.

for day in range(start_day, n_iterations + 1):
    backup_db(day - 1)                    # crash safety snapshot
    advance_clock(24h)
    set_agent_times(clock)                # FSRS decay reference

    day_order = shuffle(all_personas)
    for initiator in day_order:
        partner = random_choice(others)
        exchange_log = run_exchange(initiator, partner, n_turns)
        stance_a, stance_b = annotate_many([initiator, partner], exchange_log)
        update_stances(stance_a, stance_b)
        tracker.log(day, ...)

    write_checkpoint(day)                 # once per day, not per exchange

Contrast with the old LODAS approach: one random pair per iteration (some agents could act zero times in a day, others multiple times). The day-based loop ensures symmetric agent participation.

Exchange

An exchange is a multi-turn dialogue between two agents. n_turns is sampled uniformly from [MIN_TURNS, MAX_TURNS] each time. Agents alternate speaker/listener. After each turn the listener absorbs the utterance into memory.

ExchangeTurn(turn=1, speaker_id="agent_001", speaker_name="Zoe",
             listener_id="agent_002", utterance="I believe...")

Annotation & Likert clamp

A separate annotator LLM reads only each agent's own utterances from an exchange and outputs a stance label. It does not see the other agent's text (bias prevention).

The output is clamped to ±1 from the agent's prior stance:

new_stance = parse_likert(raw)
clamped_score = max(prev.score - 1, min(prev.score + 1, new_stance.score))
return LikertStance.from_score(clamped_score)

Without this clamp, agents jump 4 points in 2 turns (observed in early runs). The clamp enforces the incremental opinion change assumption that underpins the EPJ dynamics model.

Memory absorption

For KG conditions, GhostMemory.absorb() calls extract_triplets_batch() once per active dimension. Triplets go into the agent's GhostKG graph. On the next exchange, get_context() retrieves the highest-retrievability triplets and injects them into the agent's system prompt.

Triplet structure:

Triplet(subject="immigrants", predicate="contribute to", object="local tax base",
        dimension="general", round=3)

round is the day number, used for recency filtering when retrieving context.

Crash recovery

Before each day: backup_db(day-1) snapshots the GhostKG SQLite DB. After a crash: restore_db_backup(last_completed) rolls back to a consistent state, then the loop resumes from last_completed + 1. CSV rows for completed days are never re-written.

The 4 Conditions

ConditionMemoryKG Dimensions ActiveWhat agents remember
no_kg NullMemory none Nothing. Stateless agents. LODAS baseline.
general_only GhostMemory general Facts about the topic: policies, statistics, events heard in prior exchanges.
tom_only GhostMemory tom What the agent believes about other agents' views (Theory of Mind tracking).
full_kg GhostMemory general + tom + beliefs Topic facts, others' views, and the agent's own stated positions over time.

All four conditions use the same agents, topic, RNG seed, and number of days. The only variable is what memory is active.

Running

Single condition

PYTHONDONTWRITEBYTECODE=1 \
TOPIC=immigration \
MEMORY_CONDITION=no_kg \
OUTPUT_DIR=data/immigration/no_kg \
DB_PATH=data/immigration/no_kg/simulation.db \
N_AGENTS=140 \
N_ITERATIONS=10 \
RANDOM_SEED=42 \
.venv/bin/python src/simulation.py

Full ablation (all 4 conditions + plots)

bash run_ablation.sh immigration 10

The script runs all four conditions sequentially, then generates plots into data/immigration/plots/.

Multiple seeds

bash run_ablation.sh immigration 10 42,43,44

Results go into data/immigration/seed_42/no_kg/, data/immigration/seed_43/no_kg/, etc.

Resuming a crashed run

Re-run the exact same command. load_checkpoint() reads checkpoint.json, restore_db_backup() rolls the GhostKG DB back to the last completed day, and the loop starts from last_completed + 1. No data is lost or duplicated.

Generate plots standalone

PYTHONDONTWRITEBYTECODE=1 .venv/bin/python analysis/plots.py \
  --data-dir data/immigration \
  --output-dir data/immigration/plots

No __pycache__ files

run_ablation.sh sets PYTHONDONTWRITEBYTECODE=1 as a process-level env var before Python starts. For direct runs, prefix with the env var or use python -B. The .env file also sets this, but python-dotenv loads it after the interpreter starts — the process-level prefix is authoritative.

Topics

A topic is a Python file in topics/ that defines TOPIC_LABEL and PERSONAS. No other file needs to change when you add a topic.

File structure

TOPIC_NAME  = "climate"
TOPIC_LABEL = "climate change policy"

PERSONAS = [
    {
        "agent_id":            "agent_001",
        "name":                "Jordan Wells",
        "leaning":             PoliticalLeaning.FAR_LEFT,
        "age":                 28,
        "occupation":          "climate researcher",
        "persona_description": "...",
        "initial_stance":      LikertStance.STRONGLY_AGAINST,
    },
    # ... remaining agents
]

Stance convention

Decide before writing: what does "In Favor" mean for this topic? For immigration: In Favor = supports restrictive policy. For climate: In Favor = supports strong regulation. Document it in the file header. The prompts inject the label literally — the model does not infer the domain convention.

Memory Palace

Project memory persists across sessions in ~/.claude/projects/.../memory/. The index file (MEMORY.md) loads automatically at the start of every session. It points to four files:

  • project_ghostkg_opinion_dynamics.md — research question, architecture, key decisions, current status
  • decisions_ghostkg.md — why specific choices were made (Likert clamp, NullMemory pattern, day loop, 140-agent minimum)
  • agents_team.md — when to use each agent, recommended workflows
  • user_pranav.md — Pranav's background, collaboration preferences

Decisions are saved proactively after any architectural or research choice. The "Why" is always recorded alongside the decision — without it, the memory is just a fact list.

First Results

12 agents, immigration topic, 50 iterations, llama3.2:3b. These results use the old random-pair loop (pre-Phase 1). Re-run needed with day-based loop and 140 agents.

ConditionEntropy ΔAcceptance bias P(A|Δ=0)Key signal
no_kg +0.85 +0.21 (sycophantic) Baseline. Agents shift toward speaker 21% more when stances already match.
general_only +0.72 −0.175 (reversed) Factual memory inverts the bias. Agents resist when they recall contradicting facts.
tom_only +1.19 +0.31 (amplified) Highest divergence. Tracking others' views makes agents push back harder — more friction, not less.
full_kg +0.38 +0.08 (near-zero) Most stable. All three dimensions together dampen sycophancy most effectively.
Patricia (FAR_RIGHT, initial stance +2) shifted to −2 in 2 turns under tom_only. This is the key observation that motivated the ±1 Likert clamp in Phase 1.
