GhostKG Opinion Dynamics

Extending LLM opinion dynamics with structured decaying KG memory to measure sycophancy reduction

Extends: Cau et al. 2025, EPJ Data Science (LODAS framework)

Overview

LLM agents in opinion dynamics simulations are sycophantic — they shift their stated opinion toward whoever they just spoke with, regardless of prior beliefs. This project tests whether structured knowledge graph memory (GhostKG + FSRS spaced repetition) changes that behavior.

The core claim: "Consistency-awareness mechanisms via KGs reduce intrinsic agreeableness and introduce more realistic friction into individual agent dynamics."

The system runs four ablation conditions across the same agents and topic. No-memory baseline (LODAS) vs. three KG configurations that give agents progressively more structured memory of past exchanges.

This is not a replication of the EPJ paper — model, topic, and population differ. Results compare conditions against the project's own no_kg baseline, not EPJ figures.

GhostKG and FSRS

GhostMemory wraps a ghost_kg.AgentManager. Each agent has its own knowledge graph keyed by agent_id. When a listener receives an utterance, absorb() extracts triplets and writes them via AgentManager.absorb_content().

The FSRS (Free Spaced Repetition Scheduler) algorithm scores each triplet's retrievability based on elapsed time since last review. The clock advances CLOCK_ADVANCE_HOURS per day (default 24h). At 50 iterations that is ~7 weeks of simulated time — enough for early-day memories to decay meaningfully.

Simulation Mechanics

Day-based loop

One iteration equals one day. Every agent initiates exactly one exchange per day. Order is randomized each day. This replaces the old LODAS approach where one random pair was selected per iteration (some agents could act zero times, others multiple times).

for day in range(start_day, n_iterations + 1):
    backup_db(day - 1)                    # crash safety snapshot
    advance_clock(24h)
    set_agent_times(clock)                # FSRS decay reference

    day_order = shuffle(all_personas)
    for initiator in day_order:
        partner = random_choice(all - {initiator})
        exchange_log = run_exchange(initiator, partner, n_turns, day=day)
        stance_a, stance_b = annotate_many([initiator, partner])
        update_stances(stance_a, stance_b)
        tracker.log(day, ...)

    write_checkpoint(day)                 # once per day, not per exchange

Exchange

An exchange is a multi-turn dialogue. n_turns is sampled from [MIN_TURNS, MAX_TURNS] each time. Agents alternate speaker/listener. After each turn the listener absorbs the utterance into memory.

ExchangeTurn(turn=1, speaker_id="agent_001", speaker_name="Zoe",
             listener_id="agent_002", utterance="...")

Annotation and Likert clamp

A separate annotator LLM reads only each agent's own utterances from an exchange and returns a stance label. It never sees the counterpart's text (bias prevention). The output is clamped to ±1 from prior stance:

new_stance    = parse_likert(raw)
clamped_score = max(prev.score - 1, min(prev.score + 1, new_stance.score))
return LikertStance.from_score(clamped_score)

Without this clamp, agents jump 4 points in 2 turns. This was observed in early runs (Patricia, FAR_RIGHT initial +2, shifted to −2 in 2 turns under tom_only). The clamp enforces incremental opinion change consistent with the EPJ model assumptions.

Memory absorption

For KG conditions, GhostMemory.absorb() calls extract_triplets_batch() once per active dimension. Triplets are written to the agent's GhostKG graph. On the next exchange, get_context() retrieves highest-retrievability triplets and injects them into the agent's system prompt.

Triplet(subject="immigrants", predicate="contribute to",
        object="local tax base", dimension="general", round=3)

round is the day number, used for recency filtering.

Crash recovery

Before each day: backup_db(day-1) snapshots the GhostKG SQLite DB. After a crash: restore_db_backup(last_completed) rolls back, then the loop resumes from last_completed + 1. CSV rows for completed days are never re-written.

The 4 Conditions

ConditionMemoryKG DimensionsWhat agents remember
no_kg NullMemory none Nothing. Stateless agents. LODAS baseline.
general_only GhostMemory general Topic facts: policies, statistics, events heard in prior exchanges.
tom_only GhostMemory tom What the agent believes about other agents' views (Theory of Mind).
full_kg GhostMemory general + tom + beliefs Topic facts, others' views, and the agent's own stated positions over time.

All four conditions use the same agents, topic, RNG seed, and number of days. The only variable is the memory implementation.

Running

Single condition

PYTHONDONTWRITEBYTECODE=1 \
TOPIC=immigration \
MEMORY_CONDITION=no_kg \
OUTPUT_DIR=data/immigration/no_kg \
DB_PATH=data/immigration/no_kg/simulation.db \
N_AGENTS=140 \
N_ITERATIONS=10 \
.venv/bin/python src/simulation.py

Full ablation — all 4 conditions

bash run_ablation.sh immigration 10

Runs all four conditions sequentially, then generates plots into data/immigration/plots/.

Multiple seeds

bash run_ablation.sh immigration 10 42,43,44

Resume after crash

Re-run the exact same command. load_checkpoint() reads checkpoint.json, restore_db_backup() rolls the GhostKG DB back to the last completed day, and the loop resumes from last_completed + 1.

Suppress __pycache__

run_ablation.sh sets PYTHONDONTWRITEBYTECODE=1 as a process env var before Python starts. For direct runs: prefix with the env var or use python -B. Setting it via python-dotenv is too late — the interpreter must see it before it starts.

Generate plots only

PYTHONDONTWRITEBYTECODE=1 .venv/bin/python analysis/plots.py \
  --data-dir data/immigration \
  --output-dir data/immigration/plots

Topics

A topic is a Python file in topics/ that defines TOPIC_LABEL and PERSONAS. No other file needs to change when you add one.

File structure

TOPIC_NAME  = "climate"
TOPIC_LABEL = "climate change policy"

PERSONAS = [
    {
        "agent_id":            "agent_001",
        "name":                "Jordan Wells",
        "leaning":             PoliticalLeaning.FAR_LEFT,
        "age":                 28,
        "occupation":          "climate researcher",
        "persona_description": "...",
        "initial_stance":      LikertStance.STRONGLY_AGAINST,
    },
    # ...
]

Stance convention

Decide before writing: what does "In Favor" mean for this topic? For immigration: In Favor = supports restrictive policy. For climate: In Favor = supports strong regulation. Document it in the file header — the prompts inject the label literally.

First Results

12 agents, immigration, 50 iterations, llama3.2. These use the pre-Phase-1 random-pair loop. Re-run needed with 140 agents and day-based loop.

ConditionEntropy ΔAcceptance P(A|Δ=0)Key signal
no_kg+0.85+0.21 (sycophantic)Baseline. Agents shift toward speaker 21% more when stances already match.
general_only+0.72−0.175 (reversed)Factual memory inverts the bias. Agents resist when they recall contradicting facts.
tom_only+1.19+0.31 (amplified)Highest divergence. ToM tracking amplifies friction — agents push back harder.
full_kg+0.38+0.08 (near-zero)Most stable. All three KG dimensions together dampen sycophancy most.
Patricia (FAR_RIGHT, +2 initial) shifted to −2 in 2 exchanges under tom_only. This motivated the ±1 Likert clamp in Phase 1.
