GhostKG Opinion Dynamics

Structured, decaying memory for LLM agent opinion simulations.

Overview

This project studies how per-agent knowledge graph memory changes opinion dynamics in populations of LLM agents. It extends the LODAS baseline (Cau et al. 2025, EPJ Data Science) by giving agents a structured, decaying knowledge graph built on GhostKG.

Research question: Does KG memory help agents stay in-character as opinions shift? Does memory reduce or amplify the selective-agreement effects documented in the EPJ paper?

Baseline (LODAS): Random pairwise agent exchanges. A separate annotator LLM reads each agent's utterances and assigns a 5-point Likert stance score after each exchange. No memory between exchanges.

This framework adds four ablation conditions on top of that baseline: no memory, general-fact KG only, Theory-of-Mind KG only, and full KG (all three dimensions). The only thing that changes between conditions is MEMORY_CONDITION in .env.

ConditionWhat agents remember
no_kgNothing. Pure LODAS replication.
general_onlyTopic facts extracted from conversations.
tom_onlyInferences about each other's views.
full_kgAll three KG dimensions: general + tom + beliefs.

Null Object Pattern

exchange.py always calls memory.absorb() and memory.get_context() without an if statement. NullMemory implements both as no-ops. GhostMemory does the real work. The condition is resolved once at startup in simulation.py::_build_memory(). Zero conditionals in the hot path.

How it Works

One simulation day

  1. All agents are shuffled into a random order.
  2. Each agent, in that order, initiates one multi-turn exchange with a randomly chosen partner.
  3. The exchange runs 2-5 turns (configurable). Both agents speak each turn.
  4. After each turn, KG-enabled agents extract triplets from the listener's utterances and absorb them into the listener's knowledge graph.
  5. After the exchange, the annotator reads each agent's own utterances and assigns a Likert stance score (-2 to +2). Output is clamped to +-1 from prior stance.
  6. Stances and the full transcript are logged to CSV and JSONL.
  7. After all agents have gone, one checkpoint is written.

Likert stance and the +-1 clamp

The 5-point Likert scale maps to integer scores: FAR_LEFT = -2, LEFT = -1, CENTER = 0, RIGHT = 1, FAR_RIGHT = 2.

The annotator LLM can assign any score, but the simulation clamps it:

clamped_score = max(previous.score - 1, min(previous.score + 1, annotated.score))

Agents move at most one step per day. This prevents teleportation from FAR_LEFT to FAR_RIGHT in a single exchange and matches the LODAS baseline assumption.

Flow of control

python src/simulation.py
    |
    +-- load_config()                       # config.py: reads .env, validates all fields
    |   +-- _validate_config()
    |   +-- _smoke_test_prompts()
    |
    +-- create_personas()                   # personas.py -> list[AgentPersona]
    |
    +-- load_checkpoint()                   # checkpoint.py
    |   +-- if crash found:
    |       +-- restore_db_backup()
    |       +-- rebuild_stances() from CSV
    |
    +-- _init_ghostkg()                     # KG conditions only
    |   +-- AgentManager(db_path)
    |       +-- create_agent() x N
    |
    +-- _build_memory()                     # only place that reads memory_condition
    |   +-- NullMemory()         <-- no_kg
    |   +-- GhostMemory(manager) <-- general_only / tom_only / full_kg
    |
    +-- for day in 1..N_ITERATIONS:
        +-- backup_db(day-1)                # snapshot SQLite before touching it
        +-- clock += CLOCK_ADVANCE_HOURS
        |
        +-- for initiator in shuffle(personas):
            +-- run_exchange(initiator, partner, day=day)   # exchange.py
            |   +-- for turn in 1..n_turns:
            |       +-- build prompts (persona + stance + KG context)
            |       +-- llm_call() x2 in parallel          # llm.py
            |       +-- memory.absorb(listener, round=day)  # memory.py
            |           +-- [GhostMemory] extract_triplets_batch() -> absorb_content()
            |
            +-- annotate_many([initiator, partner])         # annotator.py
            |   +-- clamp: max(prev-1, min(prev+1, new))
            |
            +-- current_stances[a] = stance_a
            +-- current_stances[b] = stance_b
            +-- tracker.log(day, ...)                       # results.csv + exchanges.jsonl

        +-- write_checkpoint(day)           # once per day, after all agents

Knowledge graph triplets

Each triplet has five fields: (subject, predicate, object, dimension, round). The dimension field is one of general, tom, or beliefs. The round field is the current day number, used for recency filtering when building context.

GhostKG applies FSRS (spaced-repetition) decay to triplet strength over simulated time. Each KG condition filters which dimensions are stored and retrieved.

4 Conditions

ConditionEnv valueKG dimensions activeContext injected
No KG no_kg none Nothing. Pure LODAS replication. Lowest latency.
General KG general_only general Topic facts the agent has heard. Who argued what policy.
ToM KG tom_only tom Theory-of-Mind inferences. What the agent believes the partner believes.
Full KG full_kg general + tom + beliefs All three dimensions, retrieved by FSRS-weighted recency.

The condition affects only what gets extracted and retrieved. Exchange mechanics, annotation, logging, and crash recovery are identical across all four conditions.

Data Storage

All output files land in OUTPUT_DIR (set in .env, default data/run_01). This directory is gitignored.

results.csv

One row per exchange. A 140-agent 10-day run produces 1400 rows.

ColumnTypeDescription
iterationintSimulation day (1-indexed)
run_idstrUUID assigned at startup, stable across resume
memory_conditionstrno_kg | general_only | tom_only | full_kg
topicstrTopic label from TOPIC_LABEL
agent_a_idstrInitiating agent ID
agent_a_stance_before_labelstrFAR_LEFT / LEFT / CENTER / RIGHT / FAR_RIGHT
agent_a_stance_before_scoreint-2 to +2
agent_a_stance_after_labelstrAfter exchange, after clamp
agent_a_stance_after_scoreint-2 to +2
agent_b_*Same fields for the responding agent
n_turnsintNumber of turns in the exchange
sim_clockISO strSimulated clock at exchange time

exchanges.jsonl

One JSON object per exchange. Contains the full transcript (all turns), both agent IDs, stances before and after, and the raw annotator output. Use for qualitative analysis and turn-level review.

simulation.db

SQLite database owned by GhostKG's AgentManager. Present only for KG conditions. Stores all triplets with FSRS metadata (stability, difficulty, last review date). One row per triplet per agent.

Backed up before each day starts: simulation_iter_N.db.bak. On crash, the backup is restored on resume to roll the KG back to a consistent state.

checkpoint.json

Written once per day, after all agents have exchanged and all rows have been logged.

{"last_completed_iteration": 3, "memory_condition": "no_kg", "run_id": "abc-123", "topic": "immigration policy"}

On startup, if this file exists, the simulation reads it, restores the DB backup, rebuilds stances from CSV rows, and resumes from day N+1.

  OUTPUT_DIR=data/test DB_PATH=data/test/sim.db \

PYTHONDONTWRITEBYTECODE=1 must be set as a shell prefix, not in .env. Python reads it before dotenv loads. Without it, __pycache__ directories appear in every module folder.

Single condition

  .venv/bin/python src/simulation.py

Crash recovery

Re-run the same command. The simulation reads checkpoint.json, restores the DB backup, rebuilds in-memory stances from results.csv, and resumes from the next day. No duplicate rows are written.

vLLM

# Terminal 1: start server
python -m vllm.entrypoints.openai.api_server \
  --model meta-llama/Llama-3.1-8B-Instruct \
  --port 8000

# Terminal 2: run simulation
LLM_PROVIDER=vllm \
LLM_MODEL=meta-llama/Llama-3.1-8B-Instruct \
LLM_BASE_URL=http://127.0.0.1:8000/v1 \
LLM_MAX_CONCURRENCY=16 \
PARALLEL_EXCHANGE_JOBS=4 \
PYTHONDONTWRITEBYTECODE=1 .venv/bin/python src/simulation.py

vLLM uses PagedAttention (non-contiguous KV cache paging, eliminates padding waste) and continuous batching (new requests fill GPU slots as old ones finish, no batch-level waiting). The PARALLEL_EXCHANGE_JOBS setting partitions agents into disjoint pairs and dispatches them concurrently within a day, fully utilizing the server.

annotator.py

Reads agent utterances from an exchange, assigns Likert stances, clamps to +-1 from prior stance.

annotate_many(exchanges, personas, previous_stances, config) -> dict[str, LikertStance] async

Annotates all agents across all exchanges in one batch. For each agent, concatenates that agent's utterances, sends to LLM, parses Likert score, applies clamp:

clamped = max(previous.score - 1, min(previous.score + 1, annotated.score))
return LikertStance.from_score(clamped)
Returns dict mapping agent_id to new LikertStance
_retry_annotation(text, previous_stance, config, max_retries) -> LikertStance async

Called when initial annotation parse fails. Retries with a clarified prompt. Applies the same +-1 clamp on success. Returns previous_stance unchanged if all retries fail.

memory.py

Defines the AgentMemory interface and both implementations. exchange.py never imports either concrete class directly.

ABC   AgentMemory

Abstract base class with two required methods:

absorb(listener: AgentPersona, utterance: str, round: int = 0) -> None async

Called after each turn with the listener and what was said to them. NullMemory ignores everything. GhostMemory calls extract_triplets_batch() and stores results in GhostKG.

get_context(agent: AgentPersona, partner: AgentPersona) -> str

Returns a formatted string injected into the agent's prompt before each turn. NullMemory returns an empty string. GhostMemory queries the GhostKG for relevant triplets and formats them.

Returns str (may be empty)
class   NullMemory(AgentMemory)

No-op implementation. Both methods do nothing. Used for no_kg. Takes no constructor arguments.

class   GhostMemory(AgentMemory)

Full GhostKG-backed implementation. Constructor takes an AgentManager and a list of active dimensions. Active dimensions are derived from memory_condition at startup.

ConditionActive dimensions
general_only["general"]
tom_only["tom"]
full_kg["general", "tom", "beliefs"]

checkpoint.py

Crash-safe persistence. Writes checkpoint.json atomically. Manages per-day DB backups and resume detection.

Writes checkpoint.json atomically (write to temp, then rename). Called once per day after all agents have exchanged and all CSV rows have been written.

load_checkpoint(config: SimulationConfig) -> int | None

Reads checkpoint.json if it exists. Returns last_completed_iteration, or None if no checkpoint is found.

Returns int (last completed day) or None
backup_db(day: int, config: SimulationConfig) -> None

Copies config.db_path to simulation_iter_{day}.db.bak. Called at the start of each day before any exchanges run. Day 0 backup captures the empty DB state.

restore_db_backup(day: int, config: SimulationConfig) -> None

Copies simulation_iter_{day}.db.bak back to config.db_path. Called on resume to roll GhostKG back to the state before the crashed day. Raises FileNotFoundError if the backup does not exist.
